    Practical #2: Introduction to Linux and HPC¶

    Introduction¶

    In this interactive tutorial, you'll learn how to navigate the Linux filesystem and utilize fundamental bash commands through detailed, hands-on examples.

    Why Learn Linux for Bioinformatics?¶

    Linux is essential in bioinformatics and computational biology for several reasons:

    1. Most bioinformatics tools are developed for Linux environments.
    2. Linux enables processing of large genomic datasets efficiently.
    3. Command-line operations can be automated to handle repetitive tasks.
    4. High-performance computing clusters (like IBEX) run on Linux.

    As you progress in infectious disease research, these skills will help you analyze sequencing data, run phylogenetic analyses, and process large datasets more effectively.

    Getting Started with the Tutorial¶

    It's best to manually type each command into the terminal to execute them. When typing, make sure to include spaces as shown since they are critical for the commands to work correctly. While copying and pasting from the tutorial content might seem quicker, typing commands yourself will help you become more comfortable with the command line interface and assist in memorizing the basic commands.

    In this tutorial, comments in the code are marked with a hash symbol. These comments, starting with #, provide a generic template of the command but are not executed by the operating system.
    Example:

    # This is a comment line showing you the generic form of the command
# The actual command you need to type and execute follows the comment

    Linux Basics¶

    Understanding the Linux File System Structure¶

    The Linux filesystem is organized like a tree but without "drives" like those in Windows systems (e.g., C: drive). The root of this tree structure is denoted by /. While it's not necessary to know the contents of every directory and subdirectory, having a grasp of this structure will aid in navigation.

    Understanding the Home Directory¶

    When you open your terminal, you begin in your home directory. To determine the location of your home directory, use the pwd command (print working directory). Executing pwd will display the absolute path to your home directory.

    Essential File Operations¶

    Accessing Tutorial Files¶

    Start by downloading the necessary tutorial materials:

    # Command to download tutorial files (replace with actual command)

    Check the contents of your home directory:

    ls

    You should see the directory named IDE_LinuxTutorial.

    Navigating and Inspecting Directory Contents¶

    Navigate into this directory:

    cd IDE_LinuxTutorial

    View the contents:

    ls

    For a more detailed listing:

    ls -lh -t
    • -l : Detailed listing
    • -h : Human-readable file sizes
    • -t : Sort by modification time

    Understanding File Permissions¶

    Example of ls -l output:

    -rw-r--r--  file.txt
drwxr-xr-x  directory/

    Explanation: - First character: file type (- for file, d for directory)
    - Next three: owner permissions (read/write/execute)
    - Next three: group permissions
    - Last three: others' permissions

    Exploring File Contents¶

    Use the more command to read a file incrementally:

    more README.md

    Other options: - head -n 15 sequence.gb : View first 15 lines
    - tail -n 10 sequence.gb : View last 10 lines

    File Manipulation (copy, move, rename)¶

    Create a temporary folder:

    mkdir temp

    Copy a file:

    cp sequence.gb backup.md

    Move a file:

    mv backup.md temp/

    Remove a file:

    rm temp/backup.md

    Navigation and Path Concepts¶

    Absolute vs Relative Paths¶

    • Absolute path: starts from root /
    • Relative path: relative to current working directory

    Navigation symbols: - . : current directory
    - .. : parent directory
    - ~ : home directory

    Directory Navigation Shortcuts¶

    • cd - : go to previous directory
    • cd ~ : go to home directory

    Creating and Deleting Content¶

    Create new files:

    touch file1.txt

    Append text to a file:

    echo "Hello World" >> file1.txt

    Overwrite file content:

    echo "New Content" > file1.txt

    Searching for Content¶

    Find a file:

    find . -name "sequence.gb"

    Use wildcards for pattern matching:

    ls *.fasta

    Working with File Content¶

    Viewing Files¶

    • more: incremental viewing
    • less: scroll up/down freely
    • cat: display entire file
    cat sequence.gb
less sequence.gb

    Analyzing File Content¶

    Count lines, words, characters:

    wc sequence.gb
wc -l sequence.gb
wc -w sequence.gb

    Search content:

    grep "chromosome" sequence.gb

    Manipulating File Content¶

    Sort and find unique entries:

    sort data.txt
sort -n numbers.txt
sort -r data.txt
uniq data.txt
uniq -c data.txt

    Text Editors¶

    Nano - Beginner-Friendly Editor¶

    Open file:

    nano file.txt

    Basic commands: - CTRL+O : Save file
    - CTRL+X : Exit
    - CTRL+K : Cut line
    - CTRL+U : Paste line

    Vim - Advanced Editor¶

    Open file:

    vim file.txt

    Modes: - Normal: navigation
    - Insert: type text (i)
    - Command: save/quit (: commands)

    Commands: - :w : save
    - :q : quit
    - :wq : save & quit
    - i : insert mode
    - ESC : normal mode

    Shell Scripting Basics¶

    Creating Your First Bash Script¶

    Create script file:

    nano check_file.sh

    Script example:

    #!/bin/bash
if [ -f "$1" ]; then
    echo "File exists."
else
    echo "File does not exist."
fi

    Make it executable:

    chmod +x check_file.sh

    Run script:

    ./check_file.sh filename.txt

    Advanced Topics: HPC Environment¶

    HPC Login and Basic Commands¶

    Connect to IBEX HPC cluster and use the module system for software management.

    SLURM Job Submission¶

    Create SLURM script:

    #!/bin/bash
#SBATCH --job-name=genome_assembly
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --cpus-per-task=16
#SBATCH --mem=32G
#SBATCH --time=12:00:00
#SBATCH --output=assembly_%j.out
#SBATCH --error=assembly_%j.err
#SBATCH --mail-user=username@example.com
#SBATCH --mail-type=BEGIN,END,FAIL
#SBATCH --partition=batch
#SBATCH --account=bioinformatics
#SBATCH --gres=gpu:1

    Submit job:

    sbatch job_script.sh

    File Transfer¶

    Using scp:

    scp local_file user@remote:/path/

    Using rsync:

    rsync -avz local_dir user@remote:/path/

    Troubleshooting and References¶

    Common Errors¶

    Error Message Likely Cause Solution
    Permission denied File lacks execute permission chmod +x filename
    Command not found Program not installed or PATH Check spelling or install
    No such file/directory Incorrect path/filename Verify with ls
    Is a directory Command used on a directory Specify a file

    Time-Saving Keyboard Shortcuts¶

    Shortcut Function
    Tab ↹ Auto-complete commands
    ↑ Recall previous commands
    Ctrl+C Interrupt/cancel command
    Ctrl+Z Suspend process
    Ctrl+L Clear terminal
    Ctrl+A Move to beginning of line
    Ctrl+E Move to end of line

    Summary of Key Commands¶

    Command Function
    cd Change directory
    ls List contents
    pwd Show current directory
    cp Copy files
    mv Move/rename files
    rm Remove files
    mkdir Create new directory
    touch Create empty file
    find Search files by name
    grep Search content in files
    nano Basic text editor
    vim Advanced text editor
    module Manage HPC software environments
    sbatch Submit SLURM job
    sinfo Show available nodes
    squeue Show current jobs
    scancel Cancel job by ID
