    IDE Course – Day 3: Artemis - Software Install¶

    Conda¶

    Overview¶

    Conda is an open-source package management and environment management system that runs on Windows, macOS, and Linux. This guide will help you install Conda through the Miniconda distribution, which provides a minimal setup for Conda.

    Installation Steps¶

    1. Download Miniconda Installer¶

    • Visit the official Miniconda download page: https://www.anaconda.com/download/success
    • Choose the appropriate installer for your operating system:
    • Windows: Miniconda3 Windows 64-bit Graphical Installer
    • macOS: Miniconda3 macOS Intel or Apple Silicon Graphical Installer
    • Linux: Miniconda3 Linux 64-bit (x86) Installer

    2. Install Miniconda¶

    On Windows/macOS: 1. Run the downloaded .exe installer (Windows) or .pkg installer (Mac). 2. Follow the installation wizard: - Accept the license agreement. - Choose an installation location (default is recommended). - Select the option to “Add Miniconda to my PATH environment variable” (recommended). 3. Complete the installation.

    On Linux: 1. Open a terminal. 2. Navigate to the folder containing the installer file. 3. Run the installer script:

    bash Miniconda3-latest-Linux-x86_64.sh
    1. Follow the prompts:
    2. Read and accept the license agreement.
    3. Confirm the installation location (default is recommended).
    4. Choose whether to initialize Conda in your shell (recommended).

    3. Verify the Installation¶

    Once installed, verify that Conda is working correctly: 1. Open a terminal or Command Prompt. 2. Check the Conda version:

    conda --version

    4. Post-Installation Configuration¶

    1. Initialize Conda for Your Shell If Conda is not automatically initialized, run:
    conda init
    1. Test Creating an Environment Create a test environment to confirm everything works:
    conda create -n test_env python=3.9
conda activate test_env

    Artemis¶

    Overview¶

    Artemis is a free genome browser and annotation tool that allows visualisation of sequence features, next generation data, and the results of analyses within the context of the sequence, including its six-frame translation. Artemis is written in Java and is available for UNIX, Macintosh, and Windows systems. It can read EMBL and GENBANK database entries or sequences in FASTA, indexed FASTA, or raw format. Other sequence features can be in EMBL, GENBANK, or GFF format.

    Installation Steps¶

    1. Create a new conda environment:
    conda create -n artemis
    1. Activate your artemis environment:
    conda activate artemis
    1. Add conda channels:
    conda config --add channels bioconda
conda config --add channels conda-forge
    1. Install Artemis:
    conda install artemis

    Verify the Installation¶

    Open Artemis by running:

    art

    Artemis¶

    Introduction¶

    Artemis is a freely available DNA visualisation and annotation tool developed by Kim Rutherford at the Sanger Institute. This software supports a variety of file formats, including simple sequence files (e.g., FASTA) and EMBL/GenBank entries, as well as the results of sequence analyses. It provides an interactive and user-friendly graphical interface for exploring genomic data. Commonly utilised by the Pathogen Genomics group, Artemis facilitates the annotation and analysis of both prokaryotic and eukaryotic genomes and allows users to visualise mapped next-generation sequencing data.

    The tool supports the simultaneous display of multiple types of information in various contexts, such as zooming in on DNA sequence motifs or zooming out to analyse gene architecture (operons) or entire chromosomes and genomes, all within a single screen. Artemis also enables users to perform in-software analyses and save the results for future use.

    This tutorial includes adapted figures and content from the Genomics Course tutorial by the Wellcome Sanger Institute, used with acknowledgement and for educational purposes.

    Aims¶

    This module is designed to help you become acquainted with the core functions of Artemis through a series of practical examples. You are encouraged to explore additional menus and functions of Artemis beyond these exercises.

    Artemis Exercise 1¶

    1. Starting up Artemis¶

    Type the following in your terminal to open Artemis:

    art

    Click OK to confirm the working directory. All necessary files are in the Day3_Artemis directory.

    2. Loading an annotation file¶

    Load the annotation file for the Salmonella Typhi chromosome into Artemis.

    3. The basics of Artemis¶

    Familiarize yourself with the Artemis interface and explore its features.

    4. Getting around in Artemis¶

    There are three main ways to navigate to a particular DNA region: - Goto menu
    - Navigator panel
    - Feature Selector

    4.1 Goto Menu¶

    Use this menu to jump to locations within a feature or the start/end of the DNA sequence. - Highlight regions, select CDS, navigate genome start/end, and zoom to selection.

    4.2 Navigator¶

    Use the Navigator panel to: 1. Jump to a specific base position. 2. Search for genes or pseudogenes. 3. Search for tRNA genes or amino acid sequences.

    Explore menus to understand additional features.

    Artemis Exercise 2¶

    Navigate to the genomic region between 670704 and 673340 (gyrA locus).

    Annotation¶

    Click on a feature to view annotation. Use Edit -> Selected Feature in Editor to see all annotations.

    Viewing Amino Acid or Protein Sequence¶

    Use View -> Bases or View -> Amino Acids to inspect sequences.

    Plots/Graphs¶

    Select View -> Feature Plots to display hydrophobicity, hydrophilicity, and coiled-coil regions. Use the Graph menu to display GC Content (%), adjusting window size for smoothing.

    Zooming and Display Tips¶

    • Disable stop codons and remove annotations for clear genome-wide views.
    • Adjust scaling for graphs using the right-click menu and set window size.

    Artemis Exercise 3¶

    Use NCBI database searches and initial gene annotation for gyrA.

    BLAST¶

    BLAST compares sequences to databases to find similarities. - BLASTX translates nucleotide sequences in all six reading frames and compares to protein databases.

    Use NCBI BLAST webpage: https://blast.ncbi.nlm.nih.gov/Blast.cgi

    BLASTP Alignment¶

    • Letters: amino acids in query and subject.
    • +: conservative substitution.
    • Gaps (- or space): insertions/deletions.
    • Identities: exact matches; positives: identities + conservative substitutions.

    Artemis Exercise 4¶

    Go to position 1334484 to examine a Salmonella pathogenicity island (SPI-2, ~21 kb).

    Key Characteristics of SPI¶

    1. Often inserted alongside stable RNAs
    2. Atypical G+C contents
    3. Carry virulence-related functions
    4. Often carry genes for transposase/integrase-like proteins
    5. Unstable and self-mobilisable
    6. Limited phylogenetic distribution

    Extracting the Region¶

    1. Create a new entry in Artemis (Create -> New Entry).
    2. Save new files:
    spi2.dna
spi2.tab

    Defining ssa Gene Family Cluster¶

    The ssa genes encode components of Type III secretion system (T3SS), essential for Salmonella virulence. 1. Copy or move selected ssa genes into a new entry. 2. Merge annotation from .tab into .dna. 3. Save the combined entry in EMBL format.

    Open the created EMBL file in Artemis for inspection.
